# Orbit v0.1 Open Source Release Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Prepare Orbit v0.1 for a credible macOS-first open-source release.

**Architecture:** Keep release-facing assets in `public/`, use Tauri's icon
generator for platform bundles, and make repository metadata and Markdown
describe only behavior supported by the current implementation.

**Tech Stack:** SVG, Tauri v2 CLI, Markdown, Cargo, npm/Vite

---

### Task 1: Create the Orbit identity

**Files:**
- Create: `public/orbit-mark.svg`
- Create: `public/orbit-app-icon.svg`
- Modify: `src-tauri/icons/*`

**Steps:**
1. Create a transparent constellation mark for GitHub and README use.
2. Create the rounded-square app icon using the same geometry.
3. Generate the Tauri platform icon set from the app icon.
4. Validate the generated SVG and raster/icon files.

### Task 2: Correct release metadata

**Files:**
- Modify: `package.json`
- Modify: `src-tauri/Cargo.toml`
- Modify: `src-tauri/tauri.conf.json`

**Steps:**
1. Replace template descriptions and authors.
2. Add repository, license, and homepage metadata where supported.
3. Use consistent Orbit capitalization.
4. Confirm no release metadata contains template placeholders.

### Task 3: Write the open-source documentation

**Files:**
- Modify: `README.md`
- Create: `CONTRIBUTING.md`
- Create: `LICENSE`

**Steps:**
1. Rewrite the README around the user problem and actual v0.1 capabilities.
2. State macOS-first support and local-data behavior plainly.
3. Document installation, usage, adapters, architecture, limitations, and
   contribution entry points.
4. Write a concise contribution guide with exact verification commands.
5. Add the MIT license for The Darts Co.

### Task 4: Capture the product

**Files:**
- Create: `public/orbit-screenshot.png`

**Steps:**
1. Launch the full Tauri app.
2. Capture a representative application screenshot.
3. Confirm the README references the final screenshot path.

### Task 5: Verify the release package

**Steps:**
1. Check all local README links and assets.
2. Run `cargo test` from `src-tauri/`.
3. Run `npm run build`.
4. Run `git diff --check`.
5. Review the final diff for unsupported claims and unrelated changes.
